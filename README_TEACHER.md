# 济幼社团活动管理系统

> 济南幼儿师范高等专科学校 · 社团活动管理与数据分析平台

---

## 系统亮点

### 1. AI 深度赋能社团管理
- **智能审批**：材料审批时 AI 自动分析内容相关度、给出星级评分和综合得分，支持一键自动通过合格材料，大幅减少人工审核工作量
- **AI 活动策划**：输入活动主题，AI 自动生成完整的活动方案（含目的、流程、预算、注意事项）
- **AI 海报生成**：输入活动信息，自动生成宣传海报图片，可直接下载使用
- **AI 照片验证**：批量验证活动照片真实性，识别重复/无关图片，杜绝虚假活动
- **AI 公告润色**：一键优化通知公告的措辞和格式
- **AI 社团健康诊断**：综合分析社团活动频率、成员活跃度、签到率等指标，生成健康报告和改进建议
- **AI 学期计划**：根据社团特点自动生成学期活动规划
- **AI 活动总结**：基于签到数据和活动内容自动生成活动总结报告

### 2. 全流程闭环管理
- 活动创建 → 签到考勤 → 材料上传 → 审批归档 → 赋分评价 → 学分认定，全链路数字化
- 每个环节都有数据留痕，可追溯、可审计、可导出

### 3. 多维度签到防伪
- **签到码签到**：社长创建活动时生成随机签到码
- **二维码扫码**：学生通过摄像头扫描二维码完成签到
- **GPS 定位验证**：签到时记录地理位置，与活动地点比对
- **教师签到签退**：指导教师独立签到签退，定位+签退码双重验证
- **IP 地址记录**：记录签到时的 IP 地址，辅助判断异常

### 4. 精细化赋分体系
- 支持按星级设置集体/个人赋分上限
- 社团可自定义覆盖赋分规则
- 三级审核流程：社长计算 → 教师审核 → 团委终审
- 自动计算最终学分，支持多社团叠加
- 赋分结果可导出，直接对接教务系统

### 5. 实时数据洞察
- 社团活跃度排行、签到热度排行
- 成员学院/班级/年级分布分析
- 低活动社团自动预警
- AI 生成数据洞察报告
- 全部数据支持 Excel 导出

### 6. 移动端全面适配
- 所有页面响应式设计，手机/平板/电脑均可正常使用
- 侧边栏自动转为抽屉式菜单
- 触摸滑动切换、大按钮触摸目标
- 签到页面移动优先设计，扫码体验流畅

## 核心业务流程

### 1. 赋分评价流程（三级审核）
```
社长按规则计算赋分
    ↓
提交指导教师审核（教师可查看明细，通过/驳回）
    ↓
教师审核通过后提交团委
    ↓
管理员团委终审（通过/驳回）
    ↓
系统自动计算最终学分（多社团可叠加）
    ↓
学生可查看个人赋分结果
```

### 2. 活动开展流程
```
社长创建签到会话（设置活动名称/时间/地点/签到方式）
    ↓
上传活动方案（可选）
    ↓
活动开始 → 学生签到（签到码/扫码） → 教师签到（定位验证）
    ↓
活动结束 → 教师签退（签退码/定位） → 社长上传活动照片和总结
    ↓
标记活动完成 → 签到数据自动计入工作量
```

### 3. 材料审批流程（AI 辅助）
```
社长上传活动材料（荣誉/参与/创新三类，支持富文本描述）
    ↓
管理员审批 → AI 自动分析：
    ├── 内容相关度评分（0-100）
    ├── 材料质量星级（1-5星）
    └── 综合得分 + 审批建议
    ↓
管理员决策：通过 / 打回（填写原因） / AI 一键自动通过
    ↓
通过的材料可一键发布为社团风采/轮播图
```

### 4. 校外活动审批流程
```
社长/教师填写申请表（活动名称/地点/日期/描述/附件）
    ↓
管理员审批 → 通过/驳回
    ↓
通过后可开展活动
```

### 5. 退社流程
```
学生提交退社申请（填写原因）
    ↓
社长/教师/管理员审批 → 同意/拒绝
    ↓
同意后自动移除成员关系
```

### 数据库特性
- **自动建表**：首次启动自动创建全部 57 张表
- **自动迁移**：新增字段通过 ALTER TABLE 自动添加，已有数据不受影响
- **零配置**：SQLite 文件数据库，无需安装数据库服务
- **索引优化**：关键字段自动建索引，查询高效

## 部署说明

### 环境要求
- Python 3.8+
- pip
- （可选）智谱/DeepSeek API Key，用于 AI 功能

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/你的用户名/club-stats.git
cd club-stats

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置 AI（可选，不配置则 AI 功能不可用）
# Windows:
set ZHIPU_API_KEY=你的智谱API Key
set DEEPSEEK_API_KEY=你的DeepSeek API Key
# Linux/Mac:
export ZHIPU_API_KEY=你的智谱API Key
export DEEPSEEK_API_KEY=你的DeepSeek API Key

# 4. 启动服务
python server.py --debug
```

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 后端框架 | Python 3 + Flask | Web 服务、API 路由、数据库操作 |
| 数据库 | SQLite | 数据存储，57 张表，零运维 |
| 前端 | HTML5 + CSS3 + JavaScript ES6+ | 页面渲染，零框架依赖 |
| AI 大模型 | 智谱 GLM-4 / DeepSeek | 聊天、审批、生成、分析 |
| 数据可视化 | Chart.js 4.4 | 签到统计、成员分析、财务趋势 |
| 富文本编辑 | Quill.js 2.0 | 通告编辑、风采描述 |
| 二维码 | jsQR + qrcode.js | 扫码签到、二维码生成 |
| Markdown 编辑 | EasyMDE | 活动总结、公告编辑 |
| 文件处理 | Python内置 + Pillow | 文件上传、图片裁剪、头像处理 |

## 安全设计

| 措施 | 说明 |
|------|------|
| 密码加密 | werkzeug generate_password_hash / check_password_hash |
| Session 认证 | Flask session + SECRET_KEY 签名 |
| 角色鉴权 | 每个路由检查用户角色，未授权返回 403 |
| 文件隔离 | 上传文件按社团/类型分目录存储，随机 UUID 命名 |
| SQL 参数化 | 所有数据库操作使用参数化查询，防止 SQL 注入 |
| XSS 防护 | 前端输出使用 esc() 转义，富文本内容服务端过滤 |
| API Key 隔离 | AI API Key 通过环境变量配置，不硬编码在代码中 |
